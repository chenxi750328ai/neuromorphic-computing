# 类脑计算项目 · 需求分析 v0（草案）

## 1. 背景与目标

- **总目标**：建立**类脑计算**的研发与验证能力，覆盖 **训练** 与 **推理**，并体现 **小样本学习**、**自学习（持续/在线）**、**低能耗** 等方向；**不将部署限定在某一形态**（云端、本地、嵌入式/产线等均可）。  
- **能力表述（v0）**：在可接受的工程成本下，至少形成一条可复现的「类脑/脉冲」实验链路，并与 ANN 基线在精度、数据量、能耗等维度可比。  
- **示例场景（非全部）**：自动贴膜机等工业视觉/控制——用于具象化 KPI；**项目范围以类脑能力为主，场景可增删**。  
- **硬件约束**：多平台并存——4090 侧重训练与快速迭代；Atlas 200DK A2 侧重 NPU 侧推理/转换验证；FPGA 侧重可定制加速或接口；Windows/WSL 负责统一开发与协调。

## 2. 干系人与假设

| 项 | 内容 |
|----|------|
| 研发环境 | WSL + 版本锁定的 Python/CUDA/驱动 |
| 维护方 | 可复现实验、配置与日志可追溯 |
| 假设 v0 | 输入可为帧图像、特征序列或事件流；输出可为分类、检测、回归或控制量（按首选基准任务收敛） |

## 3. 功能需求（待评审打勾）

- [ ] **F1** 类脑/脉冲模型 **训练** 流水线（仿真、数据管线、实验记录）  
- [ ] **F2** **推理** 与部署：至少一条在 **GPU** 上稳定；至少一条在 **Atlas 或 FPGA** 上完成「可运行」验证（具体框架选型待定）  
- [ ] **F3** **小样本学习**：在约定基准（如 N-way K-shot 或业务小数据）上可报告指标  
- [ ] **F4** **自学习**：定义更新触发条件与安全边界（灾难性遗忘、回滚），完成最小演示或仿真  
- [ ] **F5** **能效**：在统一工况下记录功耗/能耗或相对对比（与 ANN 或不同部署对比）  
- [ ] **F6** 数据与实验管理（配置、随机种子、模型与日志归档）

## 5. 验证与验收

- **单元/集成**：核心脚本与转换链路在目标环境可重复执行。  
- **基准验收**：选定公开或自建基准，报告小样本/自学习/能效子项。  
- **场景验收（可选）**：如贴膜机——在 KPI（良率、对齐误差、节拍等）书面冻结后，作为附加验收项。  
- **回归**：模型、比特流或依赖大版本变更时重跑约定基准。
