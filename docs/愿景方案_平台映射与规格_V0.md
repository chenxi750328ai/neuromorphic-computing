## 2. 最终愿景方案（草案 · 待勾选）

对齐 README / CHARTER / GWT，愿景拆成两层（建议同时成立）：

| 层 | 内容 |
|----|------|
| **能力愿景** | 脉冲类脑闭环：训练 + 推理 + 小样本/自学习方向 + **低能耗边缘**；远期多模块 GWT（工作区广播），主体 ≠ LLM |
| **硬件愿景** | 边缘侧必须有一条 **「事件/脉冲原生」计算路径**；工位/训练侧可保留 GPU；通用边缘 NPU（昇腾类）可承担稠密/导出 OM 推理 |

**刻意不把愿景写成**：「最终板卡 BOM 必须永久是 PYNQ」或「必须等于某款忆阻器 Sci. 芯片」。  
愿景钉的是 **槽位与规格**，板卡可演进。

### 建议冻结的「架构槽位」（与实物解耦）

```text
[编排 · 金标准 · 训练]     ← 槽 T（Trainer/Orchestrator）
         │
         ▼
[通用边缘推理 / OM]       ← 槽 N（Neural Processing Unit，如昇腾）
         │  脉冲/中间态
         ▼
[事件·脉冲·LIF 原生路径] ← 槽 M（Neuromorphic / spike-native）
         │
         ▼
[验收：精度 · 延迟分段 · 能效代理指标]
```

## 3. FPGA 在解决方案中的位置（必选 A 或 B）

板上硬件**永远是真 FPGA（Xilinx PYNQ）**——问的不是「是不是假芯片」。

| 选项 | 解决方案语义 | 含义 |
|------|--------------|------|
| **A · 替身** | FPGA **填的是槽 M**：用可编程逻辑 **模拟/代理**「类脑/脉冲芯片」该干的事（事件化、LIF、稀疏脉冲处理） | 今日规格写在槽 M 上；**将来可用专用类脑 SoC 替换 PYNQ**，槽位与 bench 契约尽量不变 |
| **B · 本体** | FPGA **就是**产品/交付里的边端硬件；规格按「最终卖/部署 FPGA」写 | 不默认「再换一块脑芯」；Charter 式「SNN→FPGA」近中期落地 |

### PL 推荐（供勾选，非代签）

**推荐 A（架构语义）+ 当期实物为真 FPGA。**

理由（缩短）：

1. README 已把 Loihi / 国产类脑等列为硬件视野 → 槽 M 应允许演进。  
2. CHARTER 的 FPGA 是 **可落地的脉冲边端**，解释为「现在用真 FPGA 实现槽 M」，与 A 不矛盾。  
3. Atlas 更自然落在 **槽 N**（通用边缘 NPU），不宜把 Atlas 叙事成「类脑芯片」。  
4. 若选纯 B：也可以，但须书面承认「远期换脑芯不在规格内」，避免顶刊忆阻器叙事冲击路线。

**请总裁勾选**：

- ☐ **采用 A**（FPGA = 槽 M 替身；当前用 PYNQ 实现）  
- ☐ **采用 B**（FPGA = 最终边端本体）  
- ☐ **折中**：近中期按 B 交付验收，架构图仍保留槽 M 标签以便远期替换（写出替换门槛：____）

## 4. 当前平台如何模拟愿景（映射表）

| 愿景槽位 | 当前平台映射 | 模拟什么 | **不**假装什么 |
|----------|--------------|----------|----------------|
| **槽 T** | WSL + **4090** | 训练、ORT/PyTorch 金标准、链路编排、bench 发起 | 不是边缘量产形态 |
| **槽 N** | **Atlas 200DK** | 边缘稠密/导出后的 **SNN OM 整网推理**（已 bit-exact） | 不是「脉冲原生类脑芯」 |
| **槽 M** | **PYNQ-Z2 FPGA** | 事件/LIF/稀疏累加等 **脉冲路径**（按 A：替身脑芯；按 B：就是边端） | 不是忆阻器 SoC；不是 Loihi |

**全链路测试的意义（在语义锁死后）**：

```text
4090(槽T) ──payload──► Atlas(槽N) ──脉冲/中间态──► FPGA(槽M) ──► 回传汇总 + 金标准对照
```

测的是：**愿景三段槽位用今天的三件套能否跑通、分段延迟与精度是否达规格**——不是「证明通信 99% 所以不要 FPGA」。

双跳 daemon 的价值保留为：**槽 T↔槽 N 的编排必须服务化**（否证逐帧 SSH）。  
**槽 M 必须入链实测**，否则映射表第三行未验收。
