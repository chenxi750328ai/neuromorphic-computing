# 愿景 · 平台 · 规格 · 探索假说 V0.1

> **负责人**：陈正共 · ChenZhengGong  
> **日期**：2026-07-14（V0.1 目标对齐修订）  
> **项目性质**：**技术探索** — **未规定唯一技术路线**。  
> **对齐结论（语义）**：① 平台 = 4090+Atlas+FPGA；② FPGA 问的是方案里的**角色语义**（替身脑芯 vs 本体）；③ 要用平台对照**愿景/规格**。  
> **对齐缺口（目标）**：此前草稿把「三槽必须入链 / 先锁 A|B」写得像**已定路线**——与探索性质冲突。V0.1 改正如下。

---

## 0. 一句话（目标口径）

```text
先冻结「全链路规格」（要达到什么）
  → 在当前平台上设计对照实验（可含 / 不含 FPGA；多种切分；多种通信）
  → 用数据回答假说（能不能达成规格、代价是什么）
  → 再谈路线取舍

禁止：用半程数据（仅双跳 daemon）写「分工定稿 / 不用 FPGA / 必须 FPGA」。
```

当前平台**可以是**候选解决方案之一，也可以只是**试验床**——何者成立，**必须用「是否达成全链路规格」来判**，不能先判路线再凑数据。

## 1. 三层模型（仍成立，用途改了）

| 层 | 是什么 | 现在怎么用 |
|----|--------|------------|
| ③ 愿景 / **规格** | 能力与可度量门禁 | **探索的裁判**：所有方案同台比 |
| ② 角色语义 | 槽 T / N / M；FPGA 可解释为 A 或 B | **实验标签**，不是选型判决书 |
| ① 平台 | 4090 + Atlas + FPGA | **试验场**；也可能是最终方案的一种实现 |

---

## 2. 愿景（探索边界，非 BOM 合同）

| 层 | 内容 |
|----|------|
| **能力愿景** | 脉冲类脑闭环（训/推）、小样本与自学习方向、边缘低能耗相关验证；远期可接 GWT 多模块 |
| **探索边界** | 允许验证多条硬件叙事（通用边缘 NPU、脉冲/PL、将来类脑 SoC）；**不**预先宣布「必须永久三机」或「永久两机」 |

槽位 T/N/M 仍可作为**实验坐标系**（便于写清测的是哪一段），但：

- **不强制**「最终产品必须保留三个槽」；  
- **不强制**「槽 M 永远由 FPGA 填充」。

---

## 3. FPGA 语义（A/B）= 解释标签，不是路线票

| 标签 | 含义（实验时怎么称呼） |
|------|------------------------|
| **A · 替身** | 本轮把 FPGA **当作**「脉冲/类脑路径」的替身来测 |
| **B · 本体** | 本轮把 FPGA **当作**可能长期留下来的边端来测 |

探索阶段可以 **两套标签都跑实验**；**不要求**先勾死 A 或 B 才能开测。  
选型结论留给：**规格对比表 + 场景约束** 之后。

## 4. 当前数据能证明什么 / **不能**证明什么

| 已有证据 | 能支持的结论 | **不能**推出的结论 |
|----------|--------------|---------------------|
| 逐帧 SSH `comm_ratio≈99.9%`、e2e~6s | **「每帧 scp+ssh」不可作为量产编排** | 「以太网/通信本质不可达规格」 |
| Atlas daemon N=100，p50≈2.5ms / p95≈5.1ms | **T↔N 服务化可行** | 「双跳方案已达最终规格」；「不需要 FPGA」 |
| FPGA 单点 / 定点对照 | 脉冲定点与 Atlas pred 可一致 | 「FPGA 入链必优 / 必劣」；「没有切分方式能用」 |
| （缺失）三跳分段、多种切分、多种通信、对照规格表 | — | **任何「定稿分工」** |

## 5. 开放假说（要用规格来筛，不是用态度筛）

| ID | 假说 | 若成立意味着什么 | 缺什么数据 |
|----|------|------------------|------------|
| **H1** | **可以不用 FPGA** | 在约定**场景约束**下，仅 T+N（或仅 N）达到全链路规格 | 规格值 + 双跳（及场景全集）对照表；写清「受限场景」边界 |
| **H2** | **某种切分能使 FPGA 有用** | 某 payload / 某算子边界上，入链后精度或延迟或能效相对基线更好（或唯一能达规格） | ≥1 种切分的**链内** FPGA 实验 + 与「Atlas 全包」对照 |
| **H3** | **更强通信机制可达规格** | 不靠砍 FPGA，而靠协议/批处理/驻留/本地互联等把 e2e·抖动·带宽做到规格内 | 通信方案矩阵（SSH / TCP daemon / 其它）× 负载形态 bench |
| **H4** | 当前三件套平台本身即候选方案 | 三机（或其中稳定子集）在规格下可复现运行 | 同上 + 稳定性/运维假设写明 |

假说之间不互斥：例如 **H3 成立且 H1 在宽场景失败** → 先通信，再谈算子切分；**H2 在窄场景成立** → 规格书必须写场景边界。

## 6. 目标对齐：先定「全链路规格」草案（待改数）

规格是探索的**共同尺子**。下列为 **PoC 草案**，总裁可改数字与是否启用；**未冻结前不做路线宣布**。

| 规格项 | 草案含义 | 备注 |
|--------|----------|------|
| **G-ACC** | 相对金标准：分类一致率 / 脉冲计数容差 | 沿用 Phase4 bit-exact 或放宽为 pred 一致 |
| **G-LAT** | 端到端 p50 / p95（ms），定义是否含首包/连接 | 双跳已有参考点；三跳未测 |
| **G-COMM** | 通信：账分清、禁止每帧 SSH、端到端 95 分位÷中位≤5（倍数） | **数字与口径以** [探索规格议程 §2](./Phase4.1_探索规格与补数议程_V0.md) **为准** |
| **G-POW** | 能效代理（可选）：板卡功耗抽样或能量/帧估计 | 探索项，可后期启用 |
| **G-SCN** | **场景集**：输入率、batch、是否允许常驻服务、最大跳数 | **H1「场景受限」写在这里** |

**判决规则（建议）**：

1. 任一候选拓扑（无 FPGA / 有 FPGA×切分_i / 通信机制_j）必须交：**同一规格表上的结果行**。  
2. 「不用 FPGA」仅当：**在 G-SCN 声明的场景内**达标，并写明场景外未证。  
3. 「要用 FPGA」仅当：存在切分使规格更好或成为达标的必要件——**有数**。  
4. 「通信不够」仅当：在合理通信改数后仍系统性破 G-LAT/G-COMM——**有数**。

## 7. 平台用法（探索议程，非路线锁）

```text
平台能力：
  4090 → 训 / 金标准 / 编排 / 假说发包
  Atlas → 槽 N 候选（整网 OM）
  FPGA  → 槽 M 候选（切入或旁路对照）

实验议程（数据补齐，顺序可调）：
  E1  冻结 §6 规格数字（哪怕先标「暂定」）
  E2  通信矩阵：daemon vs 其它（服务 H3）
  E3  基线：Atlas 全包（服务 H1）
  E4  FPGA 切分矩阵：≥2 种边界 × 入链计时（服务 H2）
  E5  填规格对照表 → 才允许写「倾向性结论」
```

双跳 daemon = **E2/E3 的一块砖**，不是终审。

## 8. 与错误目标切割

| 错误目标 | 改正 |
|----------|------|
| 先锁 A/B 或「必须三槽」再测 | **先规格，假说并行测** |
| 半程数据写分工定稿 | **禁止**；只写「已否证 / 未证实」 |
| 「暂缓 FPGA」冒充架构智慧 | 改为：**H1–H4 证据不足，列入议程** |
| 探索项目假装产品 BOM 已定 | 平台可升为方案，**仅当规格判过** |
