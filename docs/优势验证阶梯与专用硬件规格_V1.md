## 2. 维一 · 用户可见规格对比（有优有劣）

### 2.0 口径

1. 规格面向用户验收，不是研发黑话。  
2. 对照：类脑网络方案 **vs** 现神经网络方案（ANN / 端侧小模型 / LLM 等）。  
3. **每条必须能填：更好 / 更差 / 持平 / 未测**——不许只留「主张优势」。  
4. 精度、性能是**门槛**：未达及格线不得吹「更省」。

### 2.1 对比表（Stage A 冻结）

| 规格 ID | 用户可见点 | 用户问什么 | 角色 | 类脑**可能更好** | 类脑**可能更差**（必须写） | 出数 |
|---------|------------|------------|------|------------------|---------------------------|------|
| **US-ACC** | **精度** | 准不准 | 门槛 | 部分时序/事件任务够用或更稳 | **同构稠密分类常不如 ANN**（MNIST：96.97% vs 98.10%） | 场景考试集并列报 |
| **US-PERF** | **性能** | 够不够快 | 门槛 | 最坏时延可承诺；事件变化响应 | **稠密任务墙钟常更慢**；tick 一多 e2e 变差 | G-LAT |
| **US-EN** | **能耗** | 电耗少不多 | 可争 | 稀疏/常静时更省 | 稠密全量时**可能不省甚至更费**（展开抹平稀疏） | E6 / G-POW |
| **US-SZ** | **模型规模** | 塞不塞得进端侧 | 可争 | 部分任务更小达及格 | 为凑精度更大/更多状态；生态远小于 ANN/LLM | footprint |
| **US-TR** | **训练成本** | 训起来贵不贵 | 可争 | 小样本局部更新理论上省 | **现证据约 3× 更慢**——默认先标可能劣势 | 训练记账 |
| **US-IN** | **推理成本** | 一笔请求多少钱 | 可争 | 常静/稀疏时费用低 | 稠密常亮可能不比小 ANN 便宜 | 费用代理 |
| **US-SL** | **自学习** | 换任务要不要回炉大模型 | 可争 | 少标注在线适应 | **开放域远弱于 LLM**；数据工程仍重 | E8 / 陈东 |

#### US-IDLE · 「没变化时少算账」

你的理解对：**核心主张是「没有（或极少）有效输入时，尽量不耗费计算」** → 省电、省流量。

| | 说明 |
|--|------|
| **用户收益** | 监控/产线大半时间画面不动：续航/电费更好，少传无效帧，有动静仍能响应 |
| **传统网络做得到吗？** | **做得到一大截。** 红外/移动侦测再唤醒；帧差判定无运动则**跳过本帧推理**；NPU/GPU sleep；云端不推流。这不是类脑专利。 |
| **类脑还想多争什么** | 不是「ANN 完全不能跳过」，而是：（1）更细粒度（神经元级稀疏，不是整帧 wake/sleep）；（2）事件相机原生对齐，少一次「满帧解码再决定跳过」；（3）必须用数字证明：相对 **「帧差/侦测唤醒的小 ANN」** 仍更省——否则优势不成立（持平或更差也要如实填） |
| **出数** | E7：待机功耗、无效时段字节数、变化响应时延；**禁止**只拿「傻瓜每帧全推 ANN」当弱对照 |
| **启用** | 无常静场景 → waive；打不过跳帧 ANN → 填更差/持平，不得改叙事 |

> **七件套**：US-ACC · US-PERF · US-EN · US-SZ · US-TR · US-IN · US-SL。US-IDLE 按产品线启用。

### 2.2 已知/倾向劣势清单（禁止删）

| 项 | 现状 | 对用户的含义 |
|----|------|--------------|
| 同构精度 | SNN &lt; ANN（约 −1.1pt） | 「一样准还更省」目前说不满 |
| 训练墙钟 | 约 3× 更慢 | 训练成本优势默认不成立 |
| 生态 | ANN/LLM 工具链远成熟 | 交付与维护可能更贵 |
| 开放域 | 不拼 LLM 生成/对话 | US-SL 只谈约定任务 |
| 现床执行 | Atlas 稠密展开 | 「无输入不计算」容易测不清或测成不省 → §3 |

### 2.3 工程项（不进用户对比表）

bit-exact、FPGA 映射、三机切分、通信编排 → 验真 / Stage-D，不是相对 ANN 的用户卖点。

## 3. 现床上怎么填结论（人话；停用「可达成性判决」）

对每一条用户规格，只答两句：

1. **现试验床测得清吗？**  
2. **测清了的话，相对对照方案：更好、更差、还是持平？**

| 填写结果 | 人话 | 下一步 |
|----------|------|--------|
| **测得清 · 更好** | 现床数字支持这类脑规格优于对照 | 可写优势；须标「现床代理意义」≠ 产品定型 |
| **测得清 · 更差或持平** | 没赢（或打平） | **如实写入**；改算法/场景或放弃卖点——禁止瞒报 |
| **测不清** | 现床把要测的东西抹平了（例：稠密展开测不出「无输入不计算」） | 进 §4 谈专用硬件；**不得**把测不清说成已有优势 |
| **未测** | 还没跑 | 不得当卖点 |

旧词「可达成性判决 / 三态能·不能·需硬件」= 上表的「测得清·更好 / 测得清·更差 / 测不清」；以后对外用本表人话。

**纪律**：测得清·更好 ≠ 上市已证；因展开测不清可进硬件门但不可改写成「算法已赢」；禁止用「本床 vs Loihi」冒充「类脑 vs ANN」；精度/性能未过门槛禁止吹能耗。

| 规格 | 测得清吗 | 相对对照（更好/更差/持平/未测） | 依赖 / 证据 |

## 4. 专用硬件方案与规格（草案，待批）

> 触发：§3 多条落入 **测不清**（且要验的是硬件语义，不是算法没调好）。

| 规格项 | 草案 | 为何要 |
|--------|------|--------|
| **H-EVT** | 原生事件/脉冲执行或可证明稀疏跳过 | 解锁 US-EN / US-IDLE 真义 |
| **H-CIM** | 近存权重 + 本地累加 | US-EN 近存语义 |
| **H-LEARN**（可选） | 片上局部可塑性 | US-SL 硬件侧 |
| **H-IO** | 事件相机/AER | 仅启用 US-IDLE 时 |
| **H-MEAS** | 板级功率可采样 | 任何能耗结论前提 |

| 方案 | In | Out | 何时 | 关系 |
|------|----|-----|------|------|
| 自研 FPGA 更大代理 | 事件队列/稀疏跳过，非点灯 | 不做真忆阻器/流片 | Stage C 且现代理不够用时 | 现 FPGA 路径升级 |
| 商用神经形态板 | 验 US-EN/IDLE/SL 硬件上限 | 不改固件 | 有渠道且多条测不清 | **维二**对照，不作 vs ANN |
| 合作 CIM 观察 | 近存评测数据 | 不承担流片 | 长周期 OR | 不阻塞 Stage D |
